The 5 Skills (Claude Code Chain)

Each skill lives in .claude/skills/. You invoke the first one — it chains to the next automatically. Or you invoke any skill individually when you only need one step.

1 Screen Interpreter screen-interpreter/SKILL.md

Trigger: "interpret this screen", "what components do I need for this", or any image/URL/text input describing a UI

What it does: Accepts a sketch photo, Figma URL, or text brief. Uses vision (for images) or Figma MCP use_figma (for URLs) to analyze the input. Outputs a structured Screen Spec — a JSON object listing every zone (header, sidebar, content areas, modals) with the component intent for each.

Real example from your wireframes:

// Screen Spec output for "Advertisers Table" screenshot { "screenType": "data-management-list", "zones": { "breadcrumb": { "type": "Breadcrumb", "items": ["Online Ad Management", "Space World"] }, "infoBanner": { "type": "Alert", "status": "info", "text": "Edit Advertiser info..." }, "toolbar": { "left": { "type": "Select", "purpose": "status-filter" }, "right": [ { "type": "Button", "variant": "outline", "label": "Bulk Upload" }, { "type": "Button", "variant": "solid", "label": "Create Advertiser" } ] }, "dataTable": { "type": "Table", "columns": ["checkbox", "Advertiser ID", "Store ID", "Advertiser Name", ...], "features": ["row-selection", "sortable", "action-column"] }, "pagination": { "type": "Pagination" } }, "modals": ["BulkUpload", "AddAdvertiser", "AddRule", "ChangeHistory"] }
Vision (native) Figma MCP (read-only) design-skill (pattern matching)
↓ passes Screen Spec JSON
2 Figma Wireframer figma-wireframer/SKILL.md

Trigger: "wireframe this in Figma", "create a Figma mockup", or auto-chained from Agent 1

What it does: Takes the Screen Spec and builds a proper Figma file via create_new_file + use_figma. Creates frames with auto-layout matching the zone structure. Uses search_design_system to pull in existing Figma components from your Design-System-OS file. Applies Osmos brand tokens (navy gradient heroes, #F2F2FC backgrounds, Montserrat headings, 10px border-radius cards).

Key detail: This agent knows your Figma design system file key (58jL2Gbe53rBhxOysvHM82) and searches it before creating anything from scratch. If a component exists in the library, it imports it via importComponentByKeyAsync.

Output: Figma file URL + PNG screenshot + component map (which Figma component was used where)

create_new_file use_figma search_design_system osmos-brand
↓ passes Figma URL + component map
3 React Implementer react-implementer/SKILL.md

Trigger: "implement this screen", "build the React for this", or auto-chained from Agent 2

What it does: Takes the Screen Spec + component map and generates production React code using only @rishikeshjoshi-morpheus/ui components. Zero inline styles. All compound components use dot notation. Wraps in OsmosProvider.

Example output for your Advertiser flow:

import { Table, Modal, Button, Input, Select, Badge, Checkbox, Alert, Breadcrumb, Pagination, FileUpload, Steps, Timeline, Tag, Switch } from '@rishikeshjoshi-morpheus/ui'; export default function AdvertisersPage() { const [showBulkUpload, setShowBulkUpload] = useState(false); const [showAddAdvertiser, setShowAddAdvertiser] = useState(false); return ( <Box> <Breadcrumb.Root> <Breadcrumb.List> <Breadcrumb.Item> <Breadcrumb.Link>Online Ad Management</Breadcrumb.Link> </Breadcrumb.Item> <Breadcrumb.Item>Space World</Breadcrumb.Item> </Breadcrumb.List> </Breadcrumb.Root> <Alert status="info">Edit Advertiser info...</Alert> <Flex justify="space-between" p="4"> <Button variant="outline" onClick={() => setShowBulkUpload(true)}> Bulk Upload </Button> <Button colorPalette="green" onClick={() => setShowAddAdvertiser(true)}> Create Advertiser </Button> </Flex> <Table.Root>...</Table.Root> {/* Full implementation with all modals */} </Box> ); }

Key rules baked into this skill:

No <div>, <button>, <input>, <table> — always use library equivalents. No style={{}} — use Chakra style props (p="4", color="fg.muted"). Sub-components via dot notation only (Modal.Body, never import { ModalBody }).

↓ passes React code + preview screenshot
4 UX Auditor ux-auditor/SKILL.md

Trigger: "audit this screen", "check UX", "is this good enough", or auto-chained from Agent 3

What it does: Reviews the Figma screenshot + React code against the Osmos-specific UX Honeycomb. Produces a scored HTML report. For your Advertisers flow, it would flag things like:

Also generates specific fix recommendations: code patches, component swaps, flow restructuring suggestions.

↓ passes audit report + fix recommendations
5 Orchestrator design-orchestrator/SKILL.md

Trigger: "build this screen end to end", "full pipeline", or invoked when you want the whole chain

What it does: Spins up Agents 1→4 in sequence using Claude Code's Agent tool. If Agent 4's audit returns any dimension below B, it routes the fix back to the right upstream agent. Packages the final deliverables.

How it chains in Claude Code:

// Inside design-orchestrator/SKILL.md — the chain logic // Step 1: Interpret the input Agent(skill: "screen-interpreter", input: user_image_or_text) → screen_spec.json // Step 2: Build Figma wireframe Agent(skill: "figma-wireframer", input: screen_spec.json) → figma_url + component_map // Step 3: Generate React code Agent(skill: "react-implementer", input: screen_spec + component_map) → AdvertisersPage.jsx + preview // Step 4: Audit UX Agent(skill: "ux-auditor", input: screenshot + code + spec) → audit_report.html // Step 5: Fix loop (if any dimension < B) if audit.lowest_score < "B": route fixes → re-run affected agent → re-audit
